# コミットメッセージ規約

## 形式

```
<type>: <説明>
```

## type一覧

| type | 用途 |
|------|------|
| feat | 新機能追加 |
| fix | バグ修正 |
| docs | ドキュメント変更 |
| remove | 機能削除 |
| refactor | リファクタリング |
| test | テスト追加・修正 |

## ルール
- 説明は日本語
- 簡潔に（1行で完結）
- 対象範囲を明確に

## 例
- `feat: 課題分類リストの分離と担当者グループ化`
- `fix: サマリー表の行の順番を詳細セクションと一致させる`
- `docs: AWS設定ガイドを追加`
