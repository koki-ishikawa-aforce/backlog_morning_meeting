# 議事録テンプレート

このドキュメントは、`generate-document` Lambda関数が生成する議事録セクションの構造を説明するものです。

## 議事録セクションの構造

```markdown
## 📝 議事録

### 担当者名

#### ⚠️ 期限超過・未完了（要対応）
- PROJECT-1: 課題名
  - **遅延理由**: <!-- 自責/社内待ち/顧客待ち/仕様変更/割り込み対応 -->
  - **ボール**: <!-- 自分/社内（誰）/顧客 -->
  - **次のアクション**: <!-- -->
  - **完了見込み**: <!-- -->

#### 🚧 期限超過・未完了（他者待ち）
- PROJECT-2: 他者待ち課題名
  - **遅延理由**: 顧客待ち
  - **ボール**: <!-- 自分/社内（誰）/顧客 -->
  - **状況**: <!-- -->

#### 📅 本日対応予定
- PROJECT-3: 本日対応予定の課題名
  - **進捗**: <!-- 進行中/完了間近/着手前/ブロック中 -->
  - **状況**: <!-- -->
  - **ボール**: <!-- 自分/社内（誰）/顧客 -->
  - **追加課題**: <!-- この課題に関連して発生した新規課題があれば記載 -->

- PROJECT-4: 今日締め切りの課題名 🔔（今日締め切り）
  - **進捗**: <!-- 進行中/完了間近/着手前/ブロック中 -->
  - **状況**: <!-- -->
  - **ボール**: <!-- 自分/社内（誰）/顧客 -->
  - **追加課題**: <!-- この課題に関連して発生した新規課題があれば記載 -->

#### 📌 その他
- **新たな課題**: <!-- 朝会中に発生した新規課題があれば記載 -->
- **他連絡事項**: <!-- その他の連絡事項があれば記載 -->

---
```

## セクション説明

### ⚠️ 期限超過・未完了（要対応）

遅延理由が「社内待ち」「顧客待ち」以外の期限超過課題。自分でアクションを取る必要がある課題が表示されます。

| 項目 | 説明 |
|------|------|
| 遅延理由 | 自責/社内待ち/顧客待ち/仕様変更/割り込み対応 から選択 |
| ボール | 自分/社内（誰）/顧客 から選択 |
| 次のアクション | 次に行うべきアクションを記載 |
| 完了見込み | 完了予定日を記載 |

### 🚧 期限超過・未完了（他者待ち）

遅延理由が「社内待ち」または「顧客待ち」の期限超過課題。他者のアクション待ちの課題が表示されます。

| 項目 | 説明 |
|------|------|
| 遅延理由 | 社内待ち/顧客待ち |
| ボール | 自分/社内（誰）/顧客 から選択 |
| 状況 | 現在の状況を記載 |

### 📅 本日対応予定

本日対応予定の課題。今日締め切りの課題には🔔マークが付きます。

| 項目 | 説明 |
|------|------|
| 進捗 | 進行中/完了間近/着手前/ブロック中 から選択 |
| 状況 | 現在の状況を記載 |
| ボール | 自分/社内（誰）/顧客 から選択 |
| 追加課題 | この課題に関連して発生した新規課題があれば記載 |

### 📌 その他

朝会中に発生した新規課題やその他の連絡事項を記載するセクション。各担当者に必ず表示されます。

| 項目 | 説明 |
|------|------|
| 新たな課題 | 朝会中に発生した新規課題があれば記載 |
| 他連絡事項 | その他の連絡事項があれば記載 |
